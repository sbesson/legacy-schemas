Introduction

This document outlines the XML Schemas created by the Open Microscopy Environment Group. Detailed information on the schemas and future development plans are available at http://www.openmicroscopy.org/site/support/file-formats. Further information on the groups work is available at http://www.openmicroscopy.org/

Auto generated documentation for the schemas is available on the development server: http://www.openmicroscopy.org/Schemas/Documentation/Generated/OME-2015-01/ome.html and http://www.openmicroscopy.org/Schemas/Documentation/Generated/OME-2011-06/OMERO.html

Active Schemas

  • BinaryFile.xsd - January 2015, version 1 [BinaryFile.xsd] The Binary File schema used to describe a file location, or the location of a fragment within a file.
  • Editor.xsd - April 2008, version 1 [Editor.xsd] The Editor schema.
  • OMERO.xsd - June 2011, version 1 [OMERO.xsd] The schema extensions used by OMERO. It is not used by the OME schema but provides a structure for data in StructuredAnnotation blocks used by OMERO.
  • ROI.xsd - January 2015, version 1 [ROI.xsd] The Region Of Interest schema.
  • SA.xsd - January 2015, version 1 [SA.xsd] The Structured Annotation schema.
  • SPW.xsd - January 2015, version 1 [SPW.xsd] The Screen/Plate/Well schema.
  • ome.xsd - January 2015, version 2 [ome.xsd] The main schema which defines the OME ontology for microscopy. This schema makes use of the others.

Legacy Schemas

  • AnalysisChain.xsd - September 2009, version 1 [AnalysisChain.xsd] The Analysis Chain schema. Analysis chains are how module outputs are connected to inputs of other modules.
  • AnalysisModule.xsd - September 2009, version 1 [AnalysisModule.xsd] The Analysis Module Library schema. Analysis modules are image analysis algorithms or, more generally, how information is generated in OME. Makes use of CLI and MLI schemas.
  • CA.xsd - September 2009, version 1 [CA.xsd] The standard Custom Attribute schema.
  • CLI.xsd - September 2009, version 1 [CLI.xsd] The Command Line Interface sub-schema.
  • DataHistory.xsd - September 2009, version 1 [DataHistory.xsd] The Data History schema. Data history describes the data dependency in OME.
  • MLI.xsd - September 2009, version 1 [MLI.xsd] The Matlab Interface sub-schema.
  • STD.xsd - September 2009, version 1 [STD.xsd] The Semantic Type definition schema. Once a Semantic Type is defined, data for the defined type can be imported into the OME server.
